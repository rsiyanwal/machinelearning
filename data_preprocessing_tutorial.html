What is .iloc?

iloc = integer-location indexing. [:, :-1] means "all rows, all columns except the last." [:, -1] means "all rows, only the last column." The trailing .values strips the Pandas wrapper and returns a raw NumPy array.

Handling Missing Values

SimpleImputer

Two cells in our dataset are NaN (Not a Number) — Salary for the Germany customer at row 4, and Age for the Spain customer at row 6.

Why NaN Breaks Models

ML algorithms perform math on your data — dot products, distances, gradient calculations. Any arithmetic involving NaN produces NaN. One missing value cascades through the entire computation and your model outputs nothing but NaN.

The NaN Cascade

44.0 + 27.0 + NaN + 38.0 = NaN   ⚠ entire column poisoned

Strategy: SimpleImputer

Instead of deleting rows with missing values (wasting real data), we impute — fill in a statistically reasonable substitute.

StrategyBest ForWhen to Avoid
meanNumerical, roughly symmetric distributionsWhen outliers skew the mean heavily
medianNumerical data with outliers (more robust)When distribution is symmetric anyway
most_frequentCategorical columnsWhen no single value clearly dominates

The fit / transform Pattern

This two-step pattern repeats in every sklearn transformer. Learn it once — apply it everywhere.

from sklearn.impute import SimpleImputer

# Step 1 — create the imputer
imputer = SimpleImputer(missing_values=np.nan, strategy='mean')

# Step 2 — fit: learn the mean of Age (col 1) and Salary (col 2)
imputer.fit(X[:, 1:3])

# Step 3 — transform: replace NaN values with the learned means
X[:, 1:3] = imputer.transform(X[:, 1:3])
# Shortcut: X[:, 1:3] = imputer.fit_transform(X[:, 1:3])

Encoding Categorical Data

OneHotEncoder · LabelEncoder

Our dataset has two text columns: Country (France/Spain/Germany) and Purchased (Yes/No). ML models are mathematical — they cannot multiply or compute distances on strings. Everything must become a number.

The Naive Encoding Problem

The obvious approach — assign France=0, Germany=1, Spain=2 — introduces a false mathematical relationship.

France = 0Germany = 1Spain = 2

⚠ This tells the model: Germany = (France + Spain) / 2   and   Spain = 2 × France. Completely false.

The Solution: One-Hot Encoding

Replace the single Country column with three binary columns — one per country. Each row gets a 1 in its country's column and 0 in all others. No false ordering. No false distances. Pure identity.

Part A — Encoding the Target Variable y with LabelEncoder

For a binary Yes/No target, LabelEncoder is appropriate. Binary values (0 and 1) carry no false ordering — the model simply learns which class means "purchased."

from sklearn.preprocessing import LabelEncoder
le = LabelEncoder()
# 'No' → 0, 'Yes' → 1 (alphabetical order)
y = le.fit_transform(y)
print(y)
Before → After

Part B — Encoding Country with OneHotEncoder + ColumnTransformer

One Country column becomes three binary columns. The ColumnTransformer applies the encoding to column 0 only, passing Age and Salary through unchanged.

Column Transformation — Country expands into 3 binary columns

from sklearn.compose import ColumnTransformer
from sklearn.preprocessing import OneHotEncoder

ct = ColumnTransformer(
    transformers=[
        ('encoder', OneHotEncoder(), [0])  # OneHotEncode column 0 (Country)
    ],
    remainder='passthrough'  # Keep Age and Salary columns unchanged
)
# np.array() converts the sparse matrix output to a dense NumPy array
X = np.array(ct.fit_transform(X))
Dummy Variable Trap

With 3 countries, knowing 2 dummy columns determines the 3rd (if is_France=0 and is_Germany=0, then is_Spain=1 by necessity). This creates multicollinearity. Modern sklearn handles this correctly by default via drop='if_binary' logic, but be aware of the concept for other tools.

Splitting into Training & Test Sets

train_test_split
The Exam Analogy

If your professor handed out the exact exam questions a week early, a student could score 100% by pure memorization — without understanding anything. To test real understanding, the exam must use new questions. ML models face the same test. We hold out data the model never sees during training, then evaluate on that held-out set. A model scoring 99% on training data but 60% on test data has overfit — it memorized, not learned.

Why random_state Matters

train_test_split shuffles the dataset randomly before dividing it. Without a fixed seed, you get a different split every run — making results impossible to reproduce, compare, or debug. random_state=1 (or any integer) locks the shuffle so everyone running the code gets identical results.

6

Feature Scaling

StandardScaler · MinMaxScaler

Look at our feature columns after encoding. Age ranges from 27–50. Salary ranges from 48,000–83,000. That is a 1,600× difference in scale.

The Dominant Feature Problem

Consider two customers — A: Age=30, Salary=50,000 and B: Age=50, Salary=51,000. The Age difference is 20. The Salary difference is 1,000. In any distance calculation, Salary dominates by 50×, making Age invisible to the algorithm. The model learns the wrong pattern — not because the data is wrong, but because the scale is misleading.

Before Scaling — Age values (27–50)
Before Scaling — Salary values (48,000–83,000)

These two features live on completely different scales — the algorithm can't fairly weigh them.

Two Scaling Strategies

StandardScaler — Standardization

z = (x − μ) / σ

Transforms each feature to mean = 0, std = 1. The result is a z-score: how many standard deviations above or below the mean.

  • z = 0 → value is at the mean
  • z = 1 → one std above mean
  • z = −2 → two stds below mean

MinMaxScaler — Normalization

x' = (x − xmin) / (xmax − xmin)

Compresses each feature into the range [0, 1]. Preserves the original distribution shape.

  • x' = 0 → minimum value
  • x' = 1 → maximum value
  • x' = 0.5 → midpoint of range
Use CaseRecommended
Most algorithms: SVM, k-NN, linear/logistic regressionStandardScaler
Neural networks with sigmoid/tanh (bounded outputs)MinMaxScaler
Data has outliers that skew the distributionStandardScaler (outliers ≠ infinite)
You need values bounded to a specific rangeMinMaxScaler

The Critical Rule: Fit on Train, Transform Both

This is the rule that catches the most beginners. The scaler must learn statistics only from training data.

✗ Wrong — Data Leakage

1. Full dataset (train + test combined)
↓
2. scaler.fit_transform(ALL data) ← leakage here
↓
3. Split into train / test
↓
4. Train model
Test data influenced the scaler. The model has "seen" the test set via its statistics. Evaluation is dishonest.

✓ Correct — No Leakage

1. Full dataset (train + test combined)
↓
2. Split into train / test first
↓
3. scaler.fit_transform(X_train only) ← learns from train
↓
4. scaler.transform(X_test) ← applies train's stats to test
Test set is truly unseen. Evaluation reflects real-world performance.
The Intuition

The test set represents future, unknown customers. Future customers cannot influence how you normalized your training data — you haven't met them yet. The correct approach simulates this by keeping test data completely isolated until after the scaler is fit.

from sklearn.preprocessing import StandardScaler

sc = StandardScaler()

# Fit on training data: learns mean and std from X_train ONLY
# Then immediately transforms X_train using those statistics
X_train = sc.fit_transform(X_train)

# Transform test data: applies the SAME mean/std learned from training
# Do NOT call fit() or fit_transform() here — that would be data leakage
X_test = sc.transform(X_test)
After Scaling — Age and Salary z-scores now live on the same scale

Both features now range roughly −2 to +2. No column dominates.

Complete Pipeline

Production Template

All six steps chained into one clean, standalone script. Use this as your starting template for every new ML project.

# ================================================================
# DATA PREPROCESSING PIPELINE — complete template
# ================================================================

# Step 1: Import Libraries
import numpy as np
import matplotlib.pyplot as plt
import pandas as pd
from sklearn.impute import SimpleImputer
from sklearn.compose import ColumnTransformer
from sklearn.preprocessing import OneHotEncoder, LabelEncoder, StandardScaler
from sklearn.model_selection import train_test_split

# Step 2: Import the Dataset
dataset = pd.read_csv('Data.csv')
X = dataset.iloc[:, :-1].values   # Features: all columns except last
y = dataset.iloc[:, -1].values    # Target: last column only

# Step 3: Handle Missing Values
imputer = SimpleImputer(missing_values=np.nan, strategy='mean')
imputer.fit(X[:, 1:3])                     # Learn means from Age and Salary
X[:, 1:3] = imputer.transform(X[:, 1:3])  # Replace NaNs with column means

# Step 4: Encode Categorical Data
ct = ColumnTransformer(
    transformers=[('encoder', OneHotEncoder(), [0])],
    remainder='passthrough'
)
X = np.array(ct.fit_transform(X))   # Country → 3 binary columns

le = LabelEncoder()
y = le.fit_transform(y)              # Yes/No → 1/0

# Step 5: Split into Training and Test Sets
X_train, X_test, y_train, y_test = train_test_split(
    X, y,
    test_size=0.2,     # 20% held out for evaluation
    random_state=1     # Fixed seed for reproducibility
)

# Step 6: Feature Scaling
sc = StandardScaler()
X_train = sc.fit_transform(X_train)  # Fit on train, transform train
X_test = sc.transform(X_test)        # Transform test using train statistics

# Data is now ready for model training
print("X_train shape:", X_train.shape)   # (8, 5)
print("X_test shape:", X_test.shape)     # (2, 5)
print("y_train:", y_train)
print("y_test:", y_test)
Output
X_train shape: (8, 5) X_test shape: (2, 5) y_train: [0 1 1 0 1 1 0 0] y_test: [0 1]
What Comes Next?

At this point, X_train and y_train are ready to pass to any sklearn model: model.fit(X_train, y_train). After training, evaluate with model.predict(X_test) and compare predictions to y_test. This is the full ML loop — and you've just completed the hardest, most error-prone part of it.
